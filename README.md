Daily Wage Service Booking Website – QA SDLC Report

Project Overview
This project focuses on testing a Daily Wage Service Booking Website where users can hire workers (e.g.,
carpenter, cleaner, electrician) for household tasks. The system includes features like user registration, login,
service search, booking, payment, and user dashboard.

SDLC Phases
1 Requirement Analysis
2 Test Planning
3 Test Design
4 Development
5 Testing
6 Deployment

Functional Modules
1 User Registration – User signup with basic details
2 Login & Logout – Authentication functionality
3 Service Search – Search for services
4 Service Details – View service info
5 Booking Module – Book service
6 Payment Module – Complete payment
7 Booking Confirmation – Confirmation after booking
8 User Dashboard / History – View past bookings

Non-Functional Requirements
1 Performance: Pages should load within 2–3 seconds
2 Usability: Simple UI and navigation
3 Reliability: No crashes and proper data saving
4 Security: Protect user and payment data
